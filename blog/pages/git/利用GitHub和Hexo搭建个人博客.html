# lxw-peter.github.io

注 ：这个是我的个人博客，之前一直不大明白怎么制作一个属于自己的博客，服务器配置啊什么的从头开始制作觉得太麻烦，后面看了一下别人的教程就能很快搭建自己的静态博客，下面是我的搭建过程。

# lxw-peter的个人博客是怎么制作出来的？

# 材料准备：

  1. git(GitHub)
  2. node.js
  3. Hexo

# 准备下锅：

  1. 安装好git后需要简单配置一下可以参考我的git学习笔记
  2. 创建一个文件夹（存放博客文件）
  3. 在GitHub上新建一个仓库，仓库名分两部分，前缀是你的GitHub用户名，后缀是github.io,我的就是lxw-peter.github.io ps: 我的已经创建过了，所以上面有提示

# 炒菜

这里笔者使用的是Hexo框架用来搭建自己的博客

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

  1. 全局安装hexo-cli

    npm install -g hexo-cli

  2. 初始化（在你之前创建的文件夹上）

     hexo init blog

  3. 进入blog文件夹

    cd blog

  4. 安装依赖

    npm install

  5. 启动本地服务器

    hexo server

  6. 浏览器输入网址 http://localhost:4000 即可查看效果

  7. 新建文章

    注意： 不要在目录source/_posts下新建文件，需要通过命令新建

        hexo new test

  8. 重新启动服务器

    1. Ctrl+C 退出服务器
    2. hexo server启动服务器

# 上菜

通过Hexo一键部署功能将本地博客部署到GitHub 上

  1. 配置文件_confim.yml文件

    注： 在deploy上有两个属性 type和repo，type 属性为 git；repo为仓库地址，这里需要注意的点是，GitHub 怎么知道你是这个GitHub账号的用户呢，所以这里需要在链接中填写你的用户名和密码,最后用@为分隔符(图上两根斜线中间的部分)。

  2. 安装hexo-deployer-git

    npm install hexo-deployer-git --save

  3. 生成静态文件

     hexo generate

  4. 一键部署

    hexo deploy

  5. 清除缓存

     hexo clean

  6. 监事文件变动

    hexo generate --watch

  7. 完成后部署

    让Hexo在生成完毕后自动部署网站

    //下面两种写法作用一样
  hexo generate --deploy
  hexo deploy --generate
//简写
  hexo g -d
  hexo d -g

# 换换口味

  1. 在GitHub上搜索hexo theme就可以搜到很多hexo的主题

  2. 选择喜欢的主题，在themes文件夹下执行克隆命令

    //depth 深度， 下载作者最新提交的版本就可以了，这样更快
  git clone url --depth 1

  3. 修改_config.yml配置

    theme: hexo-theme-yilia

  4. 重新部署

     hexo g -d

  5. 浏览器输入网址https://lxw-peter.github.io(这里换上你自己的url)查看效果
